# kuas-wifi-autologin

京都先端科学大学(KUAS)太秦キャンパスの無線LAN `kuas-wlan` に接続したときに出てくる
ログイン画面(User ID / PassWord を入力するページ)を、**自動で突破してくれる**ツールです。

一度セットアップすれば、あとは何もしなくてOK。Wi-Fiに繋がった瞬間に裏で勝手にログインします。

> 対象OS: Windows のみ(Windows 10 / 11)。PowerShellというWindows標準ツールを使います。
> 難しい操作はコピペで完結するようにしてあります。

---

## 0. 必要なもの

- Windowsパソコン
- KUAS太秦キャンパスの無線LANアカウント(いつも手入力しているUser IDとPassWord)

以上です。git や プログラミングの知識は不要です。

---

## 1. ファイルをダウンロードする

1. このページ上部にある緑色の **「Code」** ボタンをクリック
2. 出てきたメニューの一番下、**「Download ZIP」** をクリック
   （ダウンロードフォルダに `kuas-wifi-autologin-main.zip` のようなファイルが保存されます)
3. ダウンロードした zip ファイルを **右クリック → 「すべて展開」** で解凍する
4. 解凍してできたフォルダ(`kuas-wifi-autologin-main` など)を、
   わかりやすい場所(例: `C:\kuas-wifi-autologin`)に移動しておく

   > フォルダの場所はどこでも構いませんが、**この後ずっとそこに置いたままにする**必要があります。
   > (デスクトップの上でもOKですが、あとで消さないよう気をつけてください)

---

## 2. PowerShellを「管理者として」開く

1. さきほどのフォルダをエクスプローラーで開く
2. フォルダの中の何もないところで **Shiftキーを押しながら右クリック**
3. メニューから **「PowerShell ウィンドウをここで開く」**
   (表示が無い場合は「ターミナルで開く」でもOK)
4. 開いた黒い画面(PowerShell)が、これから使う場所です

   > もし手順3の項目が見当たらない場合は、スタートメニューで「PowerShell」と検索し、
   > 通常起動したあとに `cd` コマンドでフォルダへ移動してください。例:
   > ```powershell
   > cd C:\kuas-wifi-autologin
   > ```

---

## 3. 認証情報を登録する(最初の1回だけ)

PowerShellの画面に、次の1行を**そのままコピペ**してEnterキーを押します。

```powershell
powershell -ExecutionPolicy Bypass -File .\setup-credentials.ps1
```

- `User ID を入力してください` のようなメッセージが出るので、
  普段Wi-Fiログイン画面で入力している **User ID** を入力してEnter
- 続けて **PassWord** を入力してEnter
  (画面に文字は表示されませんが、ちゃんと入力されています。心配せず打ってEnter)

これで完了です。入力したパスワードは暗号化されてこのフォルダの中の
`kuas-cred.xml` というファイルに保存されます。

> 🔒 **安全性について**: このファイルは「このパソコンの、この自分のアカウント」でしか
> 中身を読み取れないように暗号化(Windows標準のDPAPIという仕組み)されます。
> 他人にファイルをコピーされても、パスワードを取り出すことはできません。

---

## 4. 自動実行の設定をする(最初の1回だけ)

続けて、PowerShellの画面に次の1行をコピペしてEnterキーを押します。

```powershell
powershell -ExecutionPolicy Bypass -File .\register-task.ps1
```

「アクセスが拒否されました」というエラーが出た場合は、PowerShellを**管理者として**
開き直す必要があります。手順2に戻り、「PowerShell ウィンドウをここで開く」の代わりに
スタートメニューでPowerShellを検索 → **右クリック →「管理者として実行」** →
`cd フォルダのパス` でこのフォルダに移動してから、もう一度同じコマンドを実行してください。

これで **Windowsのタスクスケジューラ** に登録され、以後は次のタイミングで自動的に
ログインを試みるようになります。

- Wi-Fi(`kuas-wlan`)に接続したとき
- パソコンにログオンしたとき

**セットアップはこれで全て終わりです。** これ以降は何も操作しなくて大丈夫です。

---

## 5. ちゃんと動くか確認したい場合

kuas-wlanに接続した状態で、PowerShellに次の1行を入力してEnter。

```powershell
Start-ScheduledTask -TaskName 'KUAS-WiFi-AutoLogin'
```

数秒待ってから、フォルダの中にできる `kuas-login.log` というファイルを
メモ帳などで開くと、成功したかどうかのログが記録されています。

---

## よくあるトラブル

| 症状 | 対処 |
|---|---|
| 「このシステムではスクリプトの実行が無効になっている」と出る | コマンドの `-ExecutionPolicy Bypass` を消さずに、必ずそのままコピペしてください |
| `register-task.ps1` で「アクセスが拒否されました」 | PowerShellを管理者として開き直してから再実行(手順4参照) |
| Wi-Fiに繋いでもログインされない | フォルダを移動・削除していないか確認。移動した場合は手順4をもう一度実行してください |
| パスワードを変更した | 手順3(`setup-credentials.ps1`)をもう一度実行すれば上書きされます |

## できること(技術的な補足)

- 指定したSSID(既定: `kuas-wlan`)に接続しているときだけ動作
- すでにインターネットに出られる場合は何もしない
- ポータルが必要なら自動でログインPOSTを送信(失敗時は最大5回リトライ)
- Windows タスクスケジューラに登録し、Wi-Fi接続イベント発生時とログオン時に自動実行

## 他の大学・施設のポータルで使う場合

`kuas-login.ps1` 冒頭の以下を書き換えてください。ポータルのPOSTフォームの
フィールド名(`user` / `password` 相当)が異なる場合は `$body` のキーも合わせて変更します。

```powershell
$TargetSsid = 'kuas-wlan'
$PortalUrl  = 'https://uzwlan03.kuas.ac.jp/auth/index.html/u'
```

## 注意事項

- 認証情報は平文でファイルに保存されません(DPAPI暗号化、端末・ユーザー紐付け)。
  他のPCやユーザーにはコピーしても使えません。
- ポータル証明書の検証は無効化していません。証明書エラーで失敗する場合は
  `kuas-login.log` を確認のうえ、必要な範囲で対処してください。
- 自己責任でご利用ください。ポータルの利用規約に従ってください。
