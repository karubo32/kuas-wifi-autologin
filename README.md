# kuas-wifi-autologin

京都先端科学大学(KUAS)太秦キャンパスの無線LAN `kuas-wlan` に接続したときに出てくる
ログイン画面(User ID / PassWord を入力するページ)を、**自動で突破してくれる**ツールです。

一度セットアップすれば、あとは何もしなくてOK。Wi-Fiに繋がった瞬間に裏で勝手にログインします。

> 対象OS: Windows のみ(Windows 10 / 11)。

---

## 0. 必要なもの

- Windowsパソコン
- KUAS太秦キャンパスの無線LANアカウント(いつも手入力しているUser IDとPassWord)

以上です。git やプログラミングの知識は不要です。

---

## 1. ダウンロードする

1. このページ上部にある緑色の **「Code」** ボタンをクリック
2. 出てきたメニューの一番下、**「Download ZIP」** をクリック
3. ダウンロードした zip ファイルを **右クリック → 「すべて展開」** で解凍する
4. 解凍してできたフォルダ(`kuas-wifi-autologin-main` など)を、
   わかりやすい場所(例: `C:\kuas-wifi-autologin`)に置いておく

   > このフォルダは**セットアップ後も消さずに置いたまま**にしてください。

---

## 2. `setup.bat` をダブルクリックする

フォルダの中にある **`setup.bat`** をダブルクリックしてください。それだけです。

黒い画面が開いて、案内にそって進みます。

1. 同じ黒い画面の中で、いつもWi-Fiログインで入力している
   **User ID** と **Password** を聞かれるので、そのまま入力してEnter
   (Passwordは入力しても画面に表示されませんが、ちゃんと入力されています)
   (この情報はこのパソコンの中だけで暗号化して保存され、外部には送られません)
2. 自動で続きの設定が進みます
3. 「セットアップ完了です!」と表示されたら、何かキーを押して閉じてOK

> ⚠️ Windowsが「発行元を確認できません」といった警告(SmartScreen)を出すことがあります。
> その場合は **「詳細情報」→「実行」** を選んでください。
> (このスクリプトは中身がすべてテキストで見えるので、心配な方は
> `setup.bat` / `install.ps1` をメモ帳で開いて中身を確認してから実行してください)

> ⚠️ もし途中で「アクセスが拒否されました」と出て失敗した場合:
> `setup.bat` を **右クリック →「管理者として実行」** でやり直してください。

**これでセットアップは全て終わりです。** 以降は何も操作しなくて大丈夫です。
Wi-Fi(`kuas-wlan`)に接続すると自動でログインされます。

---

## 3. ちゃんと動くか確認したい場合(任意)

kuas-wlan に接続した状態で、フォルダの中の **`test-login.bat`** をダブルクリックしてください。
手動でログインを試して、結果(成功したか)を画面に表示します。

詳しいログはフォルダ内の `kuas-login.log` にも残ります。

## よくあるトラブル

| 症状 | 対処 |
|---|---|
| Windowsに「発行元を確認できません」と警告される | 「詳細情報」→「実行」で進める(中身はテキストなので安全に確認できます) |
| `setup.bat` で「アクセスが拒否されました」と出る | `setup.bat` を右クリックして「管理者として実行」でやり直す |
| Wi-Fiに繋いでもログインされない | フォルダを移動・削除していないか確認。移動した場合は `setup.bat` をもう一度実行する |
| パスワードを変更した | `setup.bat` をもう一度実行すれば、User ID / Password を入力し直せます |

## できること(技術的な補足)

- 指定したSSID(既定: `kuas-wlan`)に接続しているときだけ動作
- すでにインターネットに出られる場合は何もしない
- ポータルが必要なら自動でログインPOSTを送信(失敗時は最大5回リトライ)
- Windows タスクスケジューラに登録し、Wi-Fi接続イベント発生時とログオン時に自動実行
- `setup.bat` は中で `install.ps1`(`setup-credentials.ps1` + `register-task.ps1` をまとめて実行)を呼んでいるだけなので、
  PowerShellに慣れている人は個別のスクリプトを直接使っても構いません

## 他の大学・施設のポータルで使う場合

`kuas-login.ps1` 冒頭の以下を書き換えてください。ポータルのPOSTフォームの
フィールド名(`user` / `password` 相当)が異なる場合は `$body` のキーも合わせて変更します。

```powershell
$TargetSsid = 'kuas-wlan'
$PortalUrl  = 'https://uzwlan03.kuas.ac.jp/auth/index.html/u'
```

## 安全性について

- **本ツールは京都先端科学大学(KUAS)による公式のものではありません。** 個人が作成した非公式ツールです。
- **User ID / Password の送信先は、大学公式のログインポータル(`uzwlan03.kuas.ac.jp`)のみです。**
  それ以外の場所(作者・第三者のサーバーなど)には一切送信されません。
  実際の送信先は [kuas-login.ps1](kuas-login.ps1) 内の `$PortalUrl` で確認できます。
- 入力した認証情報は **Windows DPAPI で暗号化**され、`kuas-cred.xml` として
  自分のPC内にのみ保存されます(このPC・このWindowsアカウント以外では復号できません)。
  平文では保存されません。
- ポータルへの通信は HTTPS(TLS)を使用し、証明書検証は無効化していません。
- すべてのソースコードはこのリポジトリで公開されており、誰でも中身を確認できます。
  実行前に `kuas-login.ps1` / `setup-credentials.ps1` / `register-task.ps1` / `install.ps1` を
  メモ帳等で開いて内容を確認することをお勧めします。
- ダウンロード元(このGitHubページ)が信頼できるものか、ご自身でご確認ください。
  同名のツールが別の場所で配布されていても、それが安全とは限りません。
- 本ツールの利用は自己責任でお願いします。作者は本ツールの利用によって生じたいかなる
  損害についても責任を負いません。大学の無線LAN利用規約の範囲内でご利用ください。
- アンインストールしたい場合は、次のコマンドでタスクを削除できます。

  ```powershell
  Unregister-ScheduledTask -TaskName 'KUAS-WiFi-AutoLogin' -Confirm:$false
  ```

  あわせて `kuas-cred.xml` を削除すれば、保存した認証情報も消去されます。
